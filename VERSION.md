# VERSION

이 문서는 `v1.0` 서비스 시작 이후의 주요 기능과 변경 사항을 기록합니다.
앞으로 `v1.x` 범위의 수정과 개선 사항을 계속 추적합니다.

## v1.0

- 정식 서비스 시작
- 사용자 로그인 및 승인 기반 접근 제어 적용
- 요약 보기, 상세 보기, 지출결의서 작성/조회, 사용자 권한 관리 화면 운영 시작
- 보고서 조회 및 기본 업무 흐름 사용이 가능한 상태로 배포

## v1.0.1

- 지출결의서 조회 화면의 PC 문서 형식을 실제 사용 문서 형식에 맞게 보정
- 지출결의서 하단 결재/서명 표시와 비고 영역을 정리
- 지출결의서 조회 화면에 인쇄 기능 추가
- 모바일에서 지출결의서 내용을 더 읽기 쉽게 카드형 상세 UI 추가
- PC와 인쇄용 지출결의서 문서 형식을 결재 영역 기준으로 다시 정리
- 모바일 전용 지출결의서 상세에서 확인 정보 영역을 줄이고 작성자 정보를 중단 영역으로 이동
- 지출결의서 목록과 모바일 상세에서 작성자 이름이 표시되도록 보정
- 모바일 전용 지출결의서 상세 UI를 서비스 기본 색상 흐름에 맞게 조정

## v1.1.0

- 자유게시판 메뉴 추가 및 로그인한 전체 사용자를 대상으로 게시글 조회/작성 기능 도입
- 자유게시판 게시글 상세 보기, 본인 글 수정/삭제, 마스터의 전체 글 삭제 권한 추가
- Supabase 자유게시판 테이블과 RLS 정책 스크립트 추가

## v1.1.1

- Vercel Web Analytics 패키지 설치 및 전역 Analytics 컴포넌트 연결
- Vercel Speed Insights 패키지 설치 및 전역 Speed Insights 컴포넌트 연결

## v1.1.2

- PC 지출결의서 인쇄 시 본문 다음 페이지부터 첨부 영수증이 이어서 출력되도록 확장
- 영수증 이미지 비율에 따라 가로형, 세로형, 정사각형 기준으로 인쇄 크기를 조절해 종이 낭비를 줄이도록 보정
- 인쇄용 영수증 영역에서 파일명을 제외하고, 이미지 준비가 끝난 뒤에만 인쇄할 수 있도록 버튼 상태를 보강

## v1.1.3

- PC 지출결의서 보기 화면의 본문 폭과 컬럼 비중을 조정해 작성일자, 작성자명, 주요 내역이 불필요하게 줄바꿈되지 않도록 개선
- 처리현황 체크 항목에서 체크한 사용자 이름은 기본 화면에서 숨기고, 마우스를 올렸을 때만 툴팁으로 확인할 수 있도록 정리
- 처리현황 각 체크 항목을 누르거나 해제할 때 확인 메시지를 먼저 보여주도록 변경해 오동작 가능성을 줄임
- 지출결의서 목록과 상세 조회 시 작성자명을 서버에서 안전하게 조합해 내려주도록 읽기 전용 API를 추가하고, 회계팀 계정에서 작성자명이 `작성자 미상`으로 보이던 문제를 수정

## v1.2.0

- 지출결의서 보기 API를 목록 조회용과 상세 조회용으로 분리하고, 서버 기준 페이지네이션을 도입해 초기 조회 부담을 줄임
- PC/모바일 지출결의서 보기 화면에 5건 단위 페이지네이션을 적용하고, 페이지 이동 및 상세 진입 시 로딩 상태가 더 분명하게 보이도록 UX를 보강
- 지출결의서 목록 캐시, 상세 캐시, 생성/수정/제출 이후 캐시 무효화 범위를 추가해 반복 조회 체감 속도를 개선하고 성능 측정용 디버그 기록 유틸을 보강
- 내 정보 저장 전용 API와 프런트엔드 프로필 저장 흐름을 추가해 이름, 소속, 연락처 수정 후 실제 데이터와 로그인 사용자 정보가 즉시 함께 갱신되도록 수정
- PC 화면 기준 공통 콘텐츠 폭을 넓혀 로그인, 요약 보기, 상세내역 보기, 지출결의서 작성, 자유게시판, 내 정보 화면의 불필요한 좌우 여백을 줄임
- 요약 보기 화면에서 조회하기 버튼을 필터와 같은 줄로 재배치하고, 사용하지 않는 주간보고서/월간보고서 버튼을 숨긴 뒤, 잔액·수입·지출·인원·헌금 명단이 한눈에 보이는 대시보드형 구조로 개편
